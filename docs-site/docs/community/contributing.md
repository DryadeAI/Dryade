# Contributing

Dryade welcomes contributions of all kinds. Our detailed [Contributing Guide](https://github.com/DryadeAI/Dryade/blob/main/CONTRIBUTING.md) covers everything you need to get started.

## Quick Links

- [Good First Issues](https://github.com/DryadeAI/Dryade/labels/good%20first%20issue) -- Great starting points for new contributors
- [Open Issues](https://github.com/DryadeAI/Dryade/issues) -- Browse all open issues
- [Pull Request Guide](https://github.com/DryadeAI/Dryade/blob/main/CONTRIBUTING.md#submitting-changes) -- How to submit your changes

## Types of Contributions

| Type | Description |
|------|-------------|
| **Code** | Bug fixes, new features, performance improvements |
| **Documentation** | Fix typos, improve guides, add tutorials |
| **Examples** | Create new quickstart projects or example integrations |
| **Translations** | Help translate the interface to more languages |
| **Bug Reports** | Report issues with clear reproduction steps |
| **Feature Ideas** | Propose improvements via [GitHub Discussions](https://github.com/DryadeAI/Dryade/discussions) |

## Development Setup

The fastest way to get a development environment running:

```bash
git clone https://github.com/YOUR-USERNAME/Dryade.git
cd Dryade
cp .env.example .env
docker compose up -d
```

For manual setup with Python and uv, see the [Contributing Guide](https://github.com/DryadeAI/Dryade/blob/main/CONTRIBUTING.md#getting-started).
